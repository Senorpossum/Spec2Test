# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security bugs seriously. Your efforts responsibly disclose via GitHub Security Advisories, helping us protect all users.

To report a security issue, please visit https://github.com/spec2test/spec2test/security/advisories/new and include all the following information:

- Description of the vulnerability
- Steps to reproduce the issue
- Affected versions
- Any suggested fixes if identified

You will receive an acknowledgment within 48 hours.

## Security Best Practices

When using Spec2Test:

1. Never commit API keys or sensitive credentials
2. Review generated tests before committing to version control
3. Keep dependencies updated via `npm update`
4. Use in isolated environments when analyzing untrusted codebases
